FOUR MODES

四大生成模式

T2V / I2V / R2V / V2V 覆盖从零创作到精准编辑的全链路。

T2V
✍️
文生视频
输入：自然语言提示词 · 输出：720P / 1080P · 3-15s
  • Prompt 是全部信息来源
  • 对光影氛围、人物真实感、风格化场景表现尤为突出
  • 最多 5000 英文 / 2500 汉字
💡 善用声音输出 · 风格要具体 · 光影要锚定 · 质感要植入
I2V
🖼️
首帧图生视频
输入：1 张图 + 可选 prompt · 画幅跟随原图
  • 首帧 = 视频第一帧画面，精确还原
  • Prompt 必须描述"从这一帧继续发生什么"
  • 不建议让人物转身 / 切换镜头视角（容易导致画面崩坏）
⚠️ 首帧是事实来源 · 只描述"动起来"，不重新构图 · 微动效要量化
R2V
🎴
多图参考生视频
输入：1-9 张参考图 + 必填 prompt
  • 图片 = 视觉参考素材，提取特征融入
  • 四种写法：单主体多角度 / 主体+场景 / 多主体交互 / 剧情分镜
  • 可自由选择画幅
📌 明确 [图N] 对应关系 · 先保留商业链路再修补质感
V2V
🎞️
视频编辑
输入：视频 + 可选参考图 + prompt
  • 四种场景：风格化改写 / 主体替换 / 场景迁移 / 多元素组合
  • 源视频是最高约束
  • 只描述需要改变的内容
🎯 明确"改什么"与"保留什么" · 超过 3 个并列改动建议分两次

LIMITS & GUARDRAILS

能力边界 & 规避策略

了解 HH 不擅长什么比知道它擅长什么更重要——这 5 条边界请提前避免触发。

!
复杂分镜与多机位切换
多镜头连续发生时容易丢失前后关系。
✅ 规避：拆分为单镜头独立生成后剪辑。
!
高速动作 / 打斗场景
赛车、跑酷、打斗容易动作断裂或穿帮。
✅ 规避：拆解为动作触点，用特写代替全景。
!
超过 10 秒的复杂提示词
单段超过 10 秒容易丢失关键阶段。
✅ 规避：控制在 5-8 秒/段，复杂叙事拆分为多段。
!
文字与 Logo 渲染
精确文字、品牌名、字幕生成不稳定。
✅ 规避：后期叠加，Prompt 中只写"标签区域干净平整"。
!
复杂液体物理
碰撞、爆炸、多人交互容易出现不符合物理规律。
✅ 规避：保留碰撞来源、杯体状态、液体落点、溅射方向。
